How Servers Handle Requests/Responses

Requests

Files (HTML, Python) when live on the web are hosted on servers. Servers are essentially large hard drives built to store files that we publish to the web. All of these servers are connected within a network (world wide web). Whenever we want to get access to these files we use the web browser and the URL which is the location of the file that we want to retrieve.

HTTP (hypertext transfer protocol) is the language used to talk to web servers. For your browser to get to your desired web address it first starts with the request line. The request line has 3 parts, method, path and version.

Method: The type of request we want to make (GET/POST).

Path: The document we are requesting (). In this URL udacity.com is the server I am are trying to contact and within that server I would like to retrieve the homepage of this nanodegree (introduction-to-programming-nanodegree--nd000).

Version: Most browsers use version 1.1, 1.0 can also be used.

Responses

A response happens after we make the request. An HTTP response produces a status line. It has the version just like the request line and they must match. The status line also has a status code and a reason phrase. If we get a status code that starts with a 2 (200) that means the request is ok and the server responds with the document.

status codes

  • 200 OK = Found the document
  • 302 found = document located somewhere else
  • 404 not found = couldn't find the document
  • 500 server error

Servers

Purpose:Respond to HTTP requests.

There are two types of responses a server will have, static and dynamic.

Static:Pre-written files like images.
Dynamic:Response that is built on the fly, web applications.

Lesson 5: Working With the App Engine

Get vs. Post

Post:

  • Parameters in the request body
  • Used for updating data
  • not ok to cache
  • OK to change the server

Get:
  • Parameters in the URL
  • Used for fetching documents
  • OK to cache
  • Max URL length(2k characters)

Lesson 8: Databases

Database

A program that stores, structures and retrieves data.

Database Transactions: ACID

Atomicity: All parts of a transaction succeed or fail together.
Consistency: The database will always be consistent.
Isolation: No transaction can interfere with another
Durability: Once the transaction is committed, it wont be lost.

HTML Templates/Repetition

Templates

Templates are used in web applications when things need to be replicated. By using templates we can reduce the amount of errors in our code. Also if we ever needed to change something that appears multiple times on our web application we can change it in one spot and have that change be made everywhere. In my web application I have a base.html file that I can insert in any of my pages. This file contains my navigation bar and my comments section. Now as I add pages to my site and my navigation gets more complex all I need to do is change the navigation list in one file and it will update everywhere.
