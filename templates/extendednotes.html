How to create a database in Python

Using sqlite3 in Python
  1. Import sqlite3 into your python file
  2. Connect to a database

    db = sqlite3.connect('data/mydb')
  3. Next, create a variable for the cursor.

    cursor = db.cursor()

    The cursor allows you to interact with the database. Creating a table, adding/removing data from the table.
- sqlite only supports 5 data types that can be used to organize data within the database.
  • null
  • integer
  • real
  • text
  • blob

How to insert data into the datbase

  1. Use your cursor to execute an insert SQL statement to create the table

    cursor.execute('''CREATE TABLE albums (title text, artist text, release_date text, publisher text, media_type text)''')
  2. Then to insert some data

    cursor.execute("INSERT INTO albums VALUES ('Glow', 'Andy Hunter', '7/24/2012', 'Xplore Records', 'MP3')")
  3. Once the data is inserted make sure to commit the changes to the database.

    conn.commit()

A Simple Guide to Five Normal Forms in Relational Database Theory

Introduction

When designing a database the normal forms defined in relational database theory represent the guidelines for record design. These rules are designed to prevent update anomalies and data inconsistencies.

First Normal Form

First normal form says that all occurrences of a record type must contain the same number of fields.

Second Normal Form

Within database tables you want all of the records to be facts about each record without having duplicate data within each row.

Example: part : warehouse : quantity : warehouse-address

This table would violate the second normal form because each part that is at the same warehouse would have the same warehouse address thus duplicating that address for every entry. To bring this back to normal form we would need to create two separate tables.

part : warehouse : quantity : warehouse-address warehouse : warehouse-address

The difference between these to examples is that the part : warehouse : quantity are all related facts to each specific part. However the warehouse address is only a fact related to the warehouse thus we need to separate warehouse-address into a separate table eliminating duplicate data.

Third Normal Form

Third normal form is violated when a non-key field is a fact about another non-key field. In that case the we would need to make separate tables.

A record is normalized if every field is either part of a the key or provides a single valued fact about the key and nothing else.

Fourth Normal Form

A record type should not contain two or more independent mult-valued facts about one entity. It also must satisfy third normal form.

Fifth Normal Form

Fifth normal form deals with cases where information can be reconstructed from smaller pieces of information that can be maintained with less redundancy. Fifth normal form generalizes cases not covered by the others.

example:
agent : company : product
smith : ford : car
smith : ford : truck
smith : GM : car
smith : GM : truck
jones : ford : car

in this case we could generate this data with normalized form by splitting it up into 3 different tables

agent : company
smith : ford
smith : GM
jones : ford

company : product
ford : car
ford : truck
GM : car
GM : truck

agent : product
smith : car
smith : truck
jones : car
