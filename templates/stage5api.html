Lesson 3: Recursion and Parallel Computing

Recursion Definitions

Two Parts:

  1. Base Case - A Starting Point
    - not defined in terms of itself
    - smallest input - already know the answer
  2. Recursive Case
    - Defined in terms of "smaller" version of itself.

Defining Procedures Recursively

- Define a procedure, factorial, that takes a natural number as its input, and returns the number of ways to arrange the input number of items.

ex:
def factorial(n):
if n == 0:
return 1
else:
return n * factorial(n-1)

Palindrome

A string that reads the same way forwards and backwards.

ex: level, any single letter, an empty string

Lesson 4: Solving Big Problems

Word Games:

When solving a problem with code it’s a good idea to break up every piece of the problem and decide what it’s going to take to solve each piece (function, string, dictionary, etc.). It’s also important to break it up to pace yourself throughout solving the problem so that you can stay focused and have short term goals.

Here's an example of breaking down the pieces needed to build a word game

  • Board - 2d array

  • Legal Play - function(pos, dir, word)
  • Score - fucntion
    • Letters - dictionary
    • Play - function
